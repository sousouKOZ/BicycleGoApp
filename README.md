# 🚲 BicycleGo（仮称）

## 📌 概要
**BicycleGo** は、  
「駐輪場が空いていない → 放置自転車につながる」問題を解決するための  
**地図 × インセンティブ（クーポン）型 駐輪支援アプリ**です。

ユーザーの現在地・駐輪場の空き状況・距離をもとに、  
**少し遠い駐輪場でも選びたくなる仕組み**を提供します。

---

## 🎯 解決したい課題
- 近くの駐輪場が満車だと、遠くまで行くのが面倒
- 結果として放置自転車が発生
- 駐輪場の利用が偏り、稼働率が最適化されていない

---

## 💡 コンセプト

## 🎟 クーポン機能（設計中）

### 基本方針
- 現在地と駐輪場の距離に応じてクーポン内容を変化
- 遠い駐輪場ほどクーポン内容を豪華にする

### クーポン発行パターン（案）

#### ① 駐輪ごとに発行
- NFC ロック完了時にクーポン発行
- ランダムで 5 種類の中から 1 つ取得
→ 利用クーポンに偏りが出ないようにしつつ、ユーザに選択させる

※ 有効期限は短め（当日〜３日（仮））
#### ② 駐輪回数による報酬
- 駐輪ごとにポイント付与
- ポイントを使って好きなクーポンと交換
→ ユーザの意思を１００％反映させることで、満足度の高いサービスにし、継続的に使ってもらう

※ 有効期限は長め（１ヶ月（仮））

## 📍 距離計算の考え方

### 現在の設計方針
- 意思決定時点の現在地をスナップショットとして使用
- リアルタイム追跡は行わない
- NFC タッチ時に距離評価を確定

### 理由
- 到着後に現在地が更新されると距離が短くなり、不整合が生じる
- UX・処理負荷・納得感のバランスを重視

---

## 🔐 駐輪ロック方式
- QR コードではなく NFC タグを使用
- 駐輪場到着時に NFC タッチ
- ロック完了と同時に以下を確定
  - 駐輪成立
  - 距離評価
  - クーポン発行

## NFCタグについて
### Human Interface Guideline(HIG)に記載されている注意点
```
HIGに記載されている注意点としては以下の3つが記載されています。

①スキャンするデバイスを接触するように促さない
NFCタグは非接触デバイスのため、端末に直接触れる必要はありません。
そのため、「スキャンする」や「かざす」などの接触する要素が含まれない言葉を使うべきと述べられています。

②ユーザーに馴染みのある言葉を使う
NFCあるいは近距離無線通信などの言葉を聞いたことがないユーザーを考慮して、ほとんどのユーザーが理解できる日常用語で説明するべきと述べられています。
例えば、「NFCタグをスキャンしてください」ではなく、「ICカードをスキャンしてください」などの表現にするなどです。

③スキャンシートの説明文は簡潔にする
Core NFCを使用した実装では、NFCを読み取る時にスキャンシートと呼ばれるハーフモーダルが表示されます。
このスキャンシートにより、タグを読み取るタイミングを明示したり、対象を明確にしたりできます。
そのため、スキャンシートにはわかりやすく簡潔で適切な文章を表示させることが推奨されています。
Qiitaより
```
